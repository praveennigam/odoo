Odoo ka Introduction

Odoo ek open-source ERP system hai jo business processes ko automate aur manage karta hai. Ye ek modular system hai jisme har ek functionality ek alag module ke roop me hoti hai.

Editions

  • Community Edition: Free, basic features, aur customization-friendly.
  • Enterprise Edition: Paid version, advanced features jaise accounting localization aur extra modules ke saath aata hai.

Architecture aur Framework

Odoo ka core architecture Model-View-Controller (MVC) pattern par based hai. Iska matlab hai:

  • Model: Data structure aur logic handle karta hai.
  • View: User interface banata hai.
  • Controller: Model aur View ke beech coordination karta hai.

Installation

  • Windows/Linux/Mac par Odoo setup karna.
  • PostgreSQL ke saath database configure karna.
  • Workspace setup karna (e.g., Visual Studio Code ya PyCharm).

Odoo Functional Overview

General Navigation

Odoo ka UI (user interface) bohot user-friendly hai aur menus aur modules ka structure well-organized hota hai. Kuch important functionalities:

  • Menus aur Modules: Odoo ke andar har ek module ka ek menu hota hai, jaise Sales, Inventory, aur CRM. Modules ka structure hierarchically hota hai jo navigation ko easy banata hai.
  • User Interface: Odoo UI responsive aur customizable hai, jisme kanban, list, aur form views milti hain.

Basic Operations (CRUD)

Odoo me CRUD operations kaafi common aur important hain:

  • Create: New record banana. Jaise, ek new customer ya invoice create karna.
  • Read: Existing records dekhna ya unka data access karna.
  • Update: Kisi record ka data modify karna, jaise ek invoice ka status change karna.
  • Delete: Records ko permanently remove karna. Jaise, galti se create hui unnecessary records ko delete karna.

Technical Overview: Odoo Framework

Odoo ka architecture modular aur MVC (Model-View-Controller) pattern par based hai. Iska matlab hai ki data ko Model ke zariye manage kiya jata hai, user interface Views ke zariye banaya jata hai, aur Controllers data aur views ke beech interaction handle karte hain.

ORM ka kaam database ko Python objects ke saath map karna hai. Models (tables) ko Python classes ke zariye define kiya jata hai, aur ORM SQL queries ko handle karta hai.

Odoo Fields - Deep Explanation

1. Char Field

Description: Ye field text data store karta hai, jaise names, titles, short descriptions, etc.

Example: Aapke name, city, email jaise fields hote hain.

name = fields.Char(string="Name", required=True)

Usage: Ye field text ke roop me ek short string store karta hai.

2. Text Field

Description: Ye field bhi text store karta hai, lekin Char field se zyada long text store kar sakta hai. Jaise ki long descriptions ya notes.

Example: description field.

description = fields.Text(string="Description")

Usage: Jab aapko ek detailed explanation ya note store karna ho.

3. Integer Field

Description: Ye field sirf integer values ko store karta hai, jaise quantity ya count.

Example: age, quantity, total_items.

quantity = fields.Integer(string="Quantity")

Usage: Ye field sirf poore numbers ko handle karta hai, aur decimal values nahi rakh sakta.

4. Float Field

Description: Ye field decimal numbers (floating point numbers) ko store karta hai, jaise prices, weights, etc.

Example: price, weight, rating.

price = fields.Float(string="Price")

Usage: Jab aapko aise numbers chahiye jo decimal values ke saath ho, jaise financial figures ya weight.

5. Boolean Field

Description: Ye field True/False ya Yes/No type of values ko store karta hai.

Example: is_active, is_approved.

is_active = fields.Boolean(string="Is Active?")

Usage: Ye field ek check-box ki tarah kaam karta hai, jisme user ko yes/no ya true/false ka option diya jata hai.

6. Date Field

Description: Ye field kisi specific date ko store karta hai, jaise birthdays, meeting dates, etc.

Example: start_date, end_date, birth_date.

start_date = fields.Date(string="Start Date")

Usage: Jab aapko sirf date store karni ho, time nahi.

7. Datetime Field

Description: Ye field date ke saath time bhi store karta hai, jaise event start time ya meeting date & time.

Example: appointment_datetime, event_start.

event_start = fields.Datetime(string="Event Start")

Usage: Jab aapko date ke saath exact time bhi store karna ho.

8. Selection Field

Description: Ye field ek dropdown menu ya list of options provide karta hai. Aap specific options ko pre-define kar sakte hain, aur user ko unme se choose karne ka option milta hai.

Example: status field with options like "Draft", "Sent", "Confirmed".

status = fields.Selection([
          ('draft', 'Draft'),
          ('sent', 'Sent'),
          ('confirmed', 'Confirmed')
      ], string="Status")

Usage: Jab aapko predefined options dikhana ho aur user ko usme se choose karne ka option dena ho.

9. Many2one Field

Description: Ye field ek relationship create karta hai jisme ek record dusre record se linked hota hai. Jaise ek sale order ka customer.

Example: customer_id, partner_id.

customer_id = fields.Many2one('res.partner', string="Customer")

Usage: Jab aapko ek record ke saath dusre record ko link karna ho (like linking a sale order with a customer).

10. One2many Field

Description: Ye field ek record ke saath multiple records ko link karta hai. Jaise ek customer ke multiple orders ho sakte hain.

Example: order_lines, sales_orders.

order_lines = fields.One2many('sale.order.line', 'order_id', string="Order Lines")

Usage: Jab ek record ka multiple records ke saath relation ho, jese ek order ke multiple line items.

A. Wizard in Odoo – Deep Explanation

Wizard ek popup form hota hai jo user se kuch inputs leta hai aur un inputs ke basis par kuch actions perform karta hai. Wizard ka data TransientModel me store hota hai, jo session ke baad automatically delete ho jata hai.

Wizard ke Features

  • Temporary Data Handle karta hai: Wizard ka data database me permanently save nahi hota.
    TransientModel unused records ko 30 minutes ke baad automatically delete kar deta hai.
  • Popup Form View Dikhata hai: Wizard ek popup ke roop me dikhai deta hai (target="new"). Ye ek specific task ko user-friendly tarike se complete karne ke liye use hota hai.
  • Input Collection aur Actions Perform karta hai: User se inputs collect karne ke baad wizard un inputs ke basis par kuch backend actions perform karta hai, jaise reports generate karna, multiple records update karna, ya filters apply karna.

Use Case: Sale Order Status Update

Is wizard ka use multiple sale orders ka status ek saath update karne ke liye kiya jayega.

Step 1: Model Define karta hai

Wizard ka model hamesha TransientModel se inherit karta hai aur isme temporary fields define kiye jate hain jo user ke inputs handle karte hain. 

  
  from odoo import models, fields, api
  
  class UpdateSaleOrderWizard(models.TransientModel):
      _name = 'update.sale.order.wizard'  < !-- Yaha pe wizard ka model define kiya gaya hai -- > 
      _description = 'Wizard to Update Sale Order Status'  < !-- Ye description hai wizard ka -- > 
  
      sale_order_ids = fields.Many2many(
          'sale.order',  < !-- Sale orders ko select karne ke liye Many2many field use kiya gaya -- > 
          string="Sale Orders",  < !-- Field ka label -- > 
          help="Un sale orders ko select karein jinhe update karna hai."  < !-- User ko field ka purpose samjhane ke liye help text -- > 
      )
  
      new_status = fields.Selection(
          [
              ('draft', 'Draft'),
              ('sent', 'Quotation Sent'),
              ('sale', 'Sale Order')  < !-- Status options jo user choose kar sakta hai -- > 
          ],
          string="New Status",  < !-- Field ka label -- > 
          required=True,  < !-- Field ko required bana diya gaya hai -- > 
          help="Naya status select karein jo sale orders ko assign karna hai."  < !-- User ko field ka purpose samjhane ke liye help text -- > 
      )
  
      def update_status(self):
          """
          Sale orders ka status update karne ke liye ye function use hota hai.
          """
          for order in self.sale_order_ids:
              order.state = self.new_status  < !-- Loop chalake selected orders ka status update kiya gaya -- >

Step 2: View Create karta hai

Wizard ka view ek form view hota hai jo popup me user inputs display karta hai. Isme buttons hote hain jo "Update" aur "Cancel" actions ko perform karte hain.

Odoo me ORM methods ka use aap record create karne, update karne, delete karne, records ko search karne, aur fetch karne ke liye karte hain.

Is example me, hum partner record ka email address update kar rahe hain.

Odoo Security: Access Rights, Record Rules, Groups, and Users (Detailed Explanation in Your Language)

Odoo me security ek important aspect hai jo aapke system ke users ko appropriate access rights dene, records ko protect karne, aur sensitive data ko secure karne me madad karta hai. Security ko manage karne ke liye, Odoo me kuch key concepts hain jaise access rights, record rules, groups, aur users.

1. Access Rights

Access rights ka use aap user groups ko define karne ke liye karte hain, jisme aap unko kisi specific model (jaise: Sales, Inventory, Accounting) pe perform karne ke liye permissions dete hain. Aapko yeh define karna hota hai ki user kis model pe Create, Read, Write, aur Delete operations perform kar sakte hain.

Types of Access Rights:

  • Create: User ko new record create karne ki permission.
  • Read: User ko record dekhne ki permission.
  • Write: User ko record ko update karne ki permission.
  • Delete: User ko record ko delete karne ki permission.

Example:

Aap ek group "Sales Manager" bana sakte hain, jo Sales Order model par Create, Read, Write, aur Delete rights rakhta ho. Yeh aap ir.model.access.csv file ke through define kar sakte hain.


access_sales_manager,access.sales.manager,model_sale_order,base.group_sale_manager,1,1,1,1

Is example me, Sales Manager group ko Sales Order model par Create, Read, Write, aur Delete rights mil rahe hain. CSV file me 1 ka matlab hai ki user ko woh permission milega.

2. Record Rules

Record rules ka use aap karte hain taaki aap specific records ko filter kar sakein jo ek user dekh sakta hai. Record rules data ko restrict karte hain, matlab agar aap chahte hain ki ek user sirf apne hi department ke records dekh sake, toh aap record rule bana sakte hain.

Record Rule Components:

  • Domain: Yeh ek filter expression hota hai jo records ko filter karta hai (jaise department_id = 1).
  • Groups: Yeh define karta hai ki yeh rule kis user group ke liye applicable hai.
  • Apply for Read/Write: Yeh decide karta hai ki rule read operations par apply hoga ya write operations par.

Example:

Maan lijiye aapko ek rule banana hai jisme Sales Manager ko sirf unhi sales orders ko dekhne ki permission mile jo unke department ke liye hain.

Apply for: Read Access

Yahaan, hum record rule me domain use kar rahe hain jisme department_id ko current user ke department ke saath match kiya jaa raha hai. Is rule ko sirf Sales Manager group pe apply kiya jaayega.

1. API Integration: External API Calls with requests Library

API Integration ka use karke aap Odoo ko external data sources ke saath interact karne ke liye set kar sakte hain. External APIs se data fetch karke, aap us data ko Odoo ke records mein store kar sakte hain.

Use Case: Maan lijiye ki aapko kisi external weather API se current weather details fetch karne hain aur unhe Odoo ke Weather model mein store karna hai.

Steps to Call External API:
  • API Call: Odoo mein requests library ko use karke external API call karte hain.
  • Data Parsing: API response ko JSON format mein parse karte hain.
  • Data Store: Data ko Odoo ke models mein store karte hain.
Code Example:

import requests
from odoo import models, fields, _

class WeatherDataFetch(models.Model):
  _name = 'weather.data.fetch'

  city = fields.Char('City')
  temperature = fields.Float('Temperature')
  description = fields.Char('Description')

  def fetch_weather(self):
      # External weather API URL
      api_url = "http://api.weatherapi.com/v1/current.json?key=YOUR_API_KEY&q=London"
      
      # API se data fetch karte hain
      response = requests.get(api_url)
      
      # Agar response successful hai (HTTP 200)
      if response.status_code == 200:
          data = response.json()  # JSON response ko parse karo
          
          # Data ko Odoo model mein save karte hain
          self.env['weather.data.fetch'].create({
              'city': data['location']['name'],
              'temperature': data['current']['temp_c'],
              'description': data['current']['condition']['text']
          })
      else:
          _logger.error("Failed to fetch weather data from the API")

11. Upgrades and Migration in Odoo

Odoo system ka upgrade aur migration process ek important task hai jo ensure karta hai ki aapka Odoo version aur modules smoothly work karein jab aap naye version pe move karte hain. Migration mainly do cheezon se related hota hai: module versioning aur database migration.

1. Module Versioning and Migration Best Practices
Module Versioning:

Module versioning ka matlab hai har module ki ek versioning scheme ko follow karna, jisse aapko code updates aur upgrades ko manage karne mein madad mile. Odoo mein har module ka version define kiya jata hai, jise aap apne custom modules ke liye set kar sakte hain.

Versioning Format: Odoo module versioning scheme usually major.minor.patch format mein hota hai.

  • Major version: Major changes jaise backward incompatible changes (e.g., data structure changes).
  • Minor version: Feature updates, enhancements, but backward compatibility maintained.
  • Patch version: Bug fixes, small improvements, and no changes to API or features.

Example - Module Versioning in __manifest__.py:


    {
        'name': 'My Custom Module',
        'version': '1.0.1',  # Major.Minor.Patch format
        'depends': ['base'],
        'data': ['views/my_custom_module_view.xml'],
        'installable': True,
    }

Benefits of Versioning:

  • Aap easily version changes track kar sakte hain.
  • Jab aap apne module ko upgrade karte hain, aap previous versions ke bugs aur fixes ko manage kar sakte hain.
  • Versioning se users ko pata chal jata hai ki kaunse updates apply kiye gaye hain.
Module Migration:

Module migration ka matlab hai apne custom modules ko naye Odoo versions ke saath compatible banana. Agar aap purane version (e.g., Odoo v14) se naye version (e.g., Odoo v16) par migrate kar rahe hain, toh aapko apne modules ko update karna padta hai.

Migration Steps:
  • Understand API Changes: Odoo ka har version apne API mein changes laata hai. Aapko apne modules ko naye APIs ke saath compatible banana padta hai.
  • Update Fields and Models: Agar database schema mein changes hue hain, toh aapko apne model definitions ko update karna hoga.
  • Data Migration: Purane version ke data ko naye version ke format mein migrate karna hota hai.

Example - Migration from Odoo v14 to v16:

Agar aapne apne module mein kuch fields add kiye hain, jaise ek custom field x_custom_field v14 mein tha, aur ab v16 mein bhi usko update karna hai, toh aapko migration script likhna padega.

Migration Script Example:


    from odoo import api, fields, models
    
    class MigrationScript(models.TransientModel):
        _name = 'custom.module.migration'
        _description = 'Migration script for custom module'
    
        def migrate_v14_to_v16(self):
            # Migration for custom field 'x_custom_field'
            # This will be called when migrating from Odoo v14 to v16
            records = self.env['custom.model'].search([('x_custom_field', '!=', False)])
            for record in records:
                # Apply necessary changes in the data, like type conversion, renaming fields, etc.
                record.write({
                    'x_custom_field': record.x_custom_field + 10  # Example modification
                })

Explanation:

  • migrate_v14_to_v16: Method is written in the custom migration script.
  • In this script, the old records are updated to the new structure or format. Here, we updated the custom field x_custom_field.
2. Database Migrations (e.g., from v14 to v16)

Database migration Odoo ke version upgrade process ka ek critical hissa hai, jisme purani database schema ko naye version ke schema ke according modify karna padta hai.

Database Migration Steps:
  • Backup the Database: Sabse pehle apne production database ka backup lena zaroori hai. Ye step aapko data loss se bachata hai. Backup karne ke liye aap pg_dump ya Odoo ke internal tools ka use kar sakte hain.
  • Check for Schema Changes: Odoo ke naye version mein schema changes ho sakte hain, jaise naye fields, tables, ya relations. Inko identify karne ke liye aapko Odoo ke release notes ya changelogs ko padhna hoga.
  • Update __manifest__.py for New Version: Module ke __manifest__.py file ko naye version ke liye update karna hoga. Jaise field definitions, dependencies, aur XML files ko naye version ke hisaab se modify karna padta hai.
  • Run Migration Commands: Odoo ke -u option ka use karke modules ko update kar sakte hain, jisse naye changes apply ho jate hain. Command: ./odoo-bin -c /etc/odoo.conf -d odoo_db -u all
  • Manual Migration Adjustments: Kabhi-kabhi, automatic migration tools sab kuch handle nahi karte, isliye aapko manually bhi data ko adjust karna padta hai, jaise field types ko change karna, purani records ko update karna, etc.
  • Test the Migration: Migration ke baad apne system ko thoroughly test karna zaroori hota hai. Check karein ki sabhi fields, views, aur reports properly migrate hue hain ya nahi.

Example - Database Migration Script:


    from odoo import api, fields, models
    
    class DatabaseMigration(models.Model):
        _name = 'custom.module.migration'
    
        @api.model
        def migrate_v14_to_v16(self):
            # Migrate records from old structure to new structure
            records = self.env['custom.model'].search([('old_field', '!=', False)])
            for record in records:
                # Migrate old data to new format
                record.write({
                    'new_field': record.old_field * 2  # Example of data transformation
                })

Explanation:

  • migrate_v14_to_v16: This function migrates the old records to the new format. It updates your custom fields, models, and relations.
3. Testing Migration:

Migration complete hone ke baad testing step sabse zaroori hai. Aapko ensure karna padta hai ki koi data loss na ho aur naye version mein sab kuch sahi se kaam karein.

Test Case Examples:

  • UI Test: Check karein ki naye features aur modules correctly render ho rahe hain.

Odoo version upgrade aur module migration ek complex process ho sakta hai, lekin agar aap careful planning aur proper versioning practices follow karte hain, toh ye process kaafi smooth ho sakta hai. Database migration aur module updates carefully handle karne se aap apne system ko stable aur future-proof bana sakte hain.
